# Medi Stream 管理端登录与欢迎页设计

## 目标

在空目录中创建一个基于 Vite、TypeScript 和原生 Web Components 的管理端前端。首期交付可用的管理员登录、登录态恢复、欢迎页、后台框架和侧边栏菜单；其余业务菜单先进入统一占位页。

## 范围

本期包含：

- 管理员账号密码登录
- Token 保存、恢复和失效处理
- 当前管理员信息加载
- 退出登录
- 可折叠侧边栏、顶栏和内容区
- 欢迎页
- 基于 Rust 后端现有模块的菜单
- 尚未开发模块的统一占位页
- 桌面端和窄屏基础适配

本期不包含管理员、用户、医院、目录、直播或权限模块的增删改查页面，也不增加新的后端接口。

## 视觉方向

采用方案 A：医疗直播运营台。

- 深蓝色侧边栏承载品牌和导航，浅色内容区承载日常操作。
- 登录页使用左右分栏：品牌信息区与账号密码表单区。
- 视觉强调稳重、清晰和高可读性，不使用复杂装饰、数据大屏风格或虚构统计。
- 欢迎页展示管理员问候、日期、模块快捷入口和建设状态。
- 使用系统字体栈，避免为首期引入字体或 UI 组件依赖。

## 页面与组件

### 登录页

登录表单包含用户名、密码和登录按钮，支持回车提交。提交期间禁用重复操作并显示加载状态；失败时显示后端返回的安全错误消息。登录成功后保存 Token，加载当前管理员信息并进入欢迎页。

### 后台框架

后台框架由侧边栏、顶栏和主内容区组成。顶栏包含侧边栏折叠按钮、当前页面标题、管理员账号和退出入口。窄屏下侧边栏作为可开合抽屉显示。

### 欢迎页

欢迎页包含简洁问候、当前日期和模块入口。首期不伪造用户量、直播数等业务统计；模块入口只表达系统能力和当前建设状态。

### 菜单

- 工作台
- 用户与机构
  - 用户管理
  - 医院管理
- 内容管理
  - 内容目录
- 直播运营
  - 直播间管理
  - 腾讯云直播
- 权限管理
  - 管理员
  - 角色
  - 权限

菜单根据 `/auth/me` 返回的权限代码控制可见性；`SUPER_ADMIN` 角色可查看全部菜单。首期没有对应业务页面的菜单进入统一占位页，不发起无意义的业务请求。

## 前端结构

采用原生 Web Components，不引入 React、Vue 或 UI 框架。

- `src/api`：统一请求、错误映射与管理员认证接口
- `src/auth`：Token 存储和当前管理员会话
- `src/router`：轻量 Hash 路由及访问控制
- `src/components`：侧边栏、顶栏和通用反馈组件
- `src/pages`：登录、欢迎和占位页面
- `src/styles`：设计令牌、全局样式及响应式规则

Web Components 通过自定义元素封装页面和框架职责。路由使用 `/admin/#/...`，负责页面选择与鉴权重定向，避免刷新子页面时依赖服务器 fallback；API 层统一添加 `Authorization: Bearer <token>`。

## API 与代理

浏览器端统一使用 `/admin/api` 前缀：

- 登录：`POST /admin/api/auth/login`
- 当前管理员：`GET /admin/api/auth/me`
- 退出：`POST /admin/api/auth/logout`

Rust 服务当前实际路由没有 `/admin/api` 前缀。本地 Vite 将 `/admin/api` 代理到 `http://127.0.0.1:8081`，并移除请求路径开头的 `/admin/api`。例如 `/admin/api/auth/login` 转发为 `http://127.0.0.1:8081/auth/login`。线上保持浏览器请求 `/admin/api/...`，由同源网关完成转发。

依赖安装和本地启动使用用户现有的 `proxy_on` shell 环境，不修改系统级代理。

## 登录态与错误处理

- Token 存入 `localStorage`，刷新页面后用 `/admin/auth/me` 恢复会话。
- 没有 Token 时访问受保护页面，直接跳转登录页。
- API 返回 401 时清理本地会话并跳转登录页。
- API 返回 403 时保留登录态并显示权限不足。
- 网络错误、非 JSON 响应和服务端错误统一转换为可读提示，不吞掉原始异常。
- 退出时调用后端注销接口；无论后端注销是否成功，都清理本地 Token，防止界面继续保持过期状态。

## 验证

- 使用正确账号可登录并进入欢迎页。
- 错误账号或密码显示错误且不会保存 Token。
- 刷新后能通过 `/auth/me` 恢复会话。
- Token 失效后自动返回登录页。
- 退出后 Token 被清除并返回登录页。
- 本地代理能把 `/admin/api/auth/login` 正确转发到 8081 的 `/auth/login`。
- 菜单折叠、分组展开、占位页跳转和窄屏侧边栏工作正常。
- 执行 TypeScript 检查和 Vite 生产构建。
- 在浏览器中检查桌面端与移动端布局、表单反馈和核心登录流程。

## 约束与兼容性

- 不改 Rust 后端路由、接口结构和鉴权协议。
- 不引入 UI 框架或额外运行时依赖。
- 不在前端保存密码。
- 首期仅根据当前 Rust 后端的 `roles` 与 `permissions` 字段控制菜单可见性，后端仍是最终权限边界。
